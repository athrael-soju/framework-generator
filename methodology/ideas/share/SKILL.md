# Share

Deliver outputs. Collect feedback. Transfer knowledge.

## Inputs

| Input | Source |
|-------|--------|
| identity_profile | Identity stage |
| deliverable | Articulate stage |
| contract | Delivery requirements |

## Process

**1. Channel** — Select delivery method
**2. Review** — Pre-delivery approval
**3. Deliver** — Execute handoff
**4. Feedback** — Collect and log
**5. Transfer** — Knowledge handoff

## Delivery Channels

| Channel | Best For |
|---------|----------|
| Presentation | Executive findings |
| Report | Detailed reference |
| Workshop | Implementation planning |
| Prototype | Technical demo |

## Output Format

```markdown
# Delivery Log | [Client] | YYYY-MM-DD

## Delivery
- Channel: X
- Date: X
- Attendees: X

## Feedback
| Item | Response | Action |
|------|----------|--------|
| Feedback | Response | Next step |

## Knowledge Transfer
- Materials provided: X
- Training: X
- Follow-up: X

## Engagement Close
- Deliverables: Complete
- Feedback: Collected
- Referral potential: High/Med/Low
```

## Quality Criteria

- [ ] Deliverable reviewed before distribution
- [ ] Channel appropriate for audience
- [ ] Feedback mechanism in place
- [ ] Knowledge transfer adequate

## Completion

Present: delivery confirmation, feedback, closure. → SPARC (referral) or Archive.
